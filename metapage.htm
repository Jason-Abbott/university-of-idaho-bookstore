Bookstore MetaPage
These pages are being served from a DELL Optiplex GXM 5100 with 32MB RAM and about 1.5GB disk space. It came with a built in 3COM something or other and some SoundBlaster 16 equivalent. I added a second NIC (Intel EtherExpress) to route AppleTalk to a lonely Macintosh.

As for software, we're now running Microsoft's Internet Information Server with Windows NT Server 3.51. I opted for IIS because it is free and has built in support for sending SQL statements through ODBC links.

The pages themselves cater to Netscape users and were written in the Programmer's File Editor (don't tell, but I've even used Internet Assistant for Word 95 a couple times). The original graphics were first created with Altamira Composer and later Caligari trueSpace2 and Photoshop. Most forms processing is handled by IIS *.idc and *.htx files, though there are a few pages where I've used Perl 5 for NT scripts or even JavaScript. I'll probably add a few ISAPI thingies as soon as I figure it out.

The product database is currently maintained in Microsoft Access 7.0 with links to the external data sources (e.g. dBase) provided by our vendors. As suggested above, the web pages are fed by SQL statements sent through an ODBC link to this database. This jargon translates into near brainless upkeep. Heck, even the people up in sales could probably do it!

This server also handles these other pages:

  • The Philosophers Page (fairly famous)
  • Conservative Chronicle Quotes (fairly unknown)
  • My own home page (fairly inflammatory)
Accesses to these pages are now logged to an Access database through ODBC. I'll have some statistics online as soon as I get around to making the logs presentable with SQL.
